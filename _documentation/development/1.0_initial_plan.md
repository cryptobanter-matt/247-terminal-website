## Framework Research & Recommendation

### Framework Comparison

#### Preact vs React vs Solid.js

| Feature | Preact | React | Solid.js |
|---------|--------|-------|----------|
| Bundle Size | ~3-4KB | ~45KB | ~22.8KB |
| Virtual DOM | Yes | Yes | No |
| React Compatibility | Near 100% | N/A | Similar syntax |
| Performance | Fast | Good | Blazing fast |
| Learning Curve | Low (React devs) | Baseline | Moderate |

#### Astro vs Next.js vs Vite

| Feature | Astro | Next.js | Vite + React |
|---------|-------|---------|--------------|
| Default JS Shipped | 0KB | 87KB+ | Full bundle |
| Architecture | Islands | Full hydration | Full hydration |
| Build Performance | Excellent | Good | Excellent |
| Static Site Support | Native | Supported | Manual |
| Best For | Content sites | Dynamic apps | SPAs |

### Recommendation: Astro + Preact

**Why Astro?**
- Zero JS by default (~40% faster load, ~90% less JS than Next.js)
- "Islands Architecture" - only hydrates interactive components
- Built on Vite = fast development experience
- Perfect for SEO-critical marketing/landing pages
- Pre-renders to static HTML by default

**Why Preact?**
- 3KB vs React's 45KB bundle size
- Same API (useState, useEffect, etc.)
- Drop-in replacement for interactive islands
- Ideal for widgets and performance-critical SPAs

### How Astro's Islands Work

```
Static content → Pure HTML (zero JS shipped)
Interactive bits → Hydrated Preact components (only where needed)
```

**Example:**
```astro
---
import Navbar from '../components/Navbar.astro'  // Static - no JS
import PricingCards from '../components/PricingCards.tsx'  // Interactive
---

<Navbar />  <!-- Pure HTML -->
<h1>Trade News Before It Goes Viral</h1>  <!-- Pure HTML -->
<PricingCards client:visible />  <!-- JS only when visible -->
```

**Component JS Requirements for 247 Terminal:**

| Component | JS Needed? | Reason |
|-----------|------------|--------|
| Hero text/images | No | Static HTML |
| Navigation links | No | Standard `<a>` tags |
| Pricing cards with hover | No | CSS handles hover states |
| "Get Access" button | No | Links to WHOP |
| Animated counter | Yes | Requires JS |
| Image carousel | Yes | Requires JS |
| Mobile menu toggle | Yes | Requires JS |

## 2025 Web Design Best Practices

### Visual Design Trends

- **Bold typography** with high contrast
- **Block-based layouts** with vibrant color contrasts
- **Micro-interactions** - hover effects, scroll-triggered animations
- **Dark mode** as primary design approach
- **Card-based layouts** - interactive, hoverable modules
- **Retro/nostalgic elements** - bold typography, grainy textures

### Performance Targets

| Metric | Target | Description |
|--------|--------|-------------|
| LCP | < 2.5s | Largest Contentful Paint |
| TTI | < 3.8s | Time to Interactive |
| FID | < 100ms | First Input Delay |
| CLS | < 0.1 | Cumulative Layout Shift |

### SaaS Landing Page Conversion Best Practices

**Benchmarks:**
- Average conversion rate: 9.5%
- Median conversion rate: 3.0%
- Top performers: 15%+

**Key Principles:**
1. **Single CTA focus** - Multiple CTAs decrease conversions by 266%
2. **Social proof** - Testimonials increase conversions by 34%
3. **Trust badges** - "No credit card required", security certs
4. **Interactive demos** - Let users experience the product
5. **Mobile-first** - 50%+ traffic is mobile
6. **Above-the-fold clarity** - All key info visible immediately
7. **Minimal form friction** - Short, easy forms

### Mobile Optimization Requirements
- Touch-friendly navigation
- Readable fonts (16px+ base)
- No intrusive pop-ups
- Fast loading (minimize heavy images)
- Thumb-friendly CTA buttons
